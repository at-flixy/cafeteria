## Технический стек (обязательно)

- Next.js 14 (App Router, RSC где уместно) + TypeScript (strict)
- Prisma ORM с SQLite (`file:./prisma/dev.db`)
- Tailwind CSS + shadcn/ui (Button, Card, Input, Dialog, Toast, Table, Badge, Tabs, Select, DropdownMenu, Sheet)
- NextAuth (Credentials provider) + bcrypt для паролей
- Zod для валидации DTO
- react-hook-form для форм

- qrcode (npm) для генерации QR-кода заказа
- lucide-react для иконок
- date-fns + локаль `ru`
- UI — на русском языке

Никакого Redux/Zustand — состояние через серверные actions и useSWR (или fetch + revalidate).

## Prisma schema (минимум, обязательно эти модели)

```prisma
enum Role { CLIENT CASHIER MANAGER ADMIN }
enum UserStatus { ACTIVE BLOCKED DELETED }
enum MenuStatus { DRAFT PUBLISHED CLOSED }
enum OrderStatus { CREATED PENDING_PAYMENT PAID READY ISSUED CANCELLED }
enum PaymentMethod { ONLINE CASH STUDENT_CARD }
enum PaymentStatus { PENDING SUCCESS FAILED REFUNDED }

model User {
  id            String   @id @default(cuid())
  fullName      String
  email         String   @unique
  passwordHash  String
  role          Role     @default(CLIENT)
  status        UserStatus @default(ACTIVE)
  createdAt     DateTime @default(now())
  orders        Order[]  @relation("client_orders")
  issuedOrders  Order[]  @relation("cashier_issued")
  notifications Notification[]
  audit         AuditLog[]
}

model Dish {
  id          String @id @default(cuid())
  name        String
  description String?
  price       Decimal @db.Decimal(10,2)
  calories    Int     @default(0)
  proteins_g  Decimal @default(0) @db.Decimal(6,2)
  fats_g      Decimal @default(0) @db.Decimal(6,2)
  carbs_g     Decimal @default(0) @db.Decimal(6,2)
  allergens   String?
  photoUrl    String?
  createdAt   DateTime @default(now())
  menuItems   MenuItem[]
}

model DailyMenu {
  id          String @id @default(cuid())
  menuDate    DateTime @unique
  status      MenuStatus @default(DRAFT)
  publishedAt DateTime?
  items       MenuItem[]
}

model MenuItem {
  id            String @id @default(cuid())
  menuId        String
  dishId        String
  price         Decimal @db.Decimal(10,2)
  availableQty  Int
  reservedQty   Int    @default(0)
  stopListFlag  Boolean @default(false)
  menu          DailyMenu @relation(fields: [menuId], references: [id])
  dish          Dish @relation(fields: [dishId], references: [id])
  orderItems    OrderItem[]
  @@unique([menuId, dishId])
}

model TimeSlot {
  id            String @id @default(cuid())
  slotDate      DateTime
  startTime     String   // "12:00"
  endTime       String   // "12:30"
  orderLimit    Int
  reservedCount Int      @default(0)
  orders        Order[]
  @@unique([slotDate, startTime])
}

model Order {
  id            String @id @default(cuid())
  orderNumber   String @unique
  clientId      String
  slotId        String
  status        OrderStatus @default(CREATED)
  totalAmount   Decimal @db.Decimal(10,2)
  createdAt     DateTime @default(now())
  issuedAt      DateTime?
  issuedById    String?
  client        User @relation("client_orders", fields: [clientId], references: [id])
  cashier       User? @relation("cashier_issued", fields: [issuedById], references: [id])
  slot          TimeSlot @relation(fields: [slotId], references: [id])
  items         OrderItem[]
  payment       Payment?
}

model OrderItem {
  id          String @id @default(cuid())
  orderId     String
  menuItemId  String
  quantity    Int
  unitPrice   Decimal @db.Decimal(10,2)
  order       Order @relation(fields: [orderId], references: [id], onDelete: Cascade)
  menuItem    MenuItem @relation(fields: [menuItemId], references: [id])
}

model Payment {
  id          String @id @default(cuid())
  orderId     String @unique
  method      PaymentMethod
  status      PaymentStatus @default(PENDING)
  providerRef String?
  paidAt      DateTime?
  order       Order @relation(fields: [orderId], references: [id])
}

model Notification {
  id        String @id @default(cuid())
  userId    String
  message   String
  type      String
  readAt    DateTime?
  createdAt DateTime @default(now())
  user      User @relation(fields: [userId], references: [id])
}

model AuditLog {
  id         String @id @default(cuid())
  actorId    String
  action     String
  entity     String
  entityId   String
  payload    String?    // JSON-строка
  createdAt  DateTime @default(now())
  actor      User @relation(fields: [actorId], references: [id])
}

model ReportSnapshot {
  id         String @id @default(cuid())
  periodFrom DateTime
  periodTo   DateTime
  type       String     // DAILY | WEEKLY
  payload    String     // JSON
  createdAt  DateTime @default(now())
}
```

Примечание: SQLite не поддерживает Decimal нативно — Prisma эмулирует. Если возникнут проблемы, замени Decimal на Float и округляй до 2 знаков на уровне приложения.

## Бизнес-логика (`src/lib/orders.ts`)

Реализовать как чистые функции, оборачивающие Prisma-транзакции.

### createOrder(clientId, slotId, items: {menuItemId, quantity}[])
Транзакция:
1. Проверить, что у каждого menuItem: stopListFlag=false, availableQty - reservedQty >= quantity → иначе OutOfStockError.
2. Проверить slot.reservedCount < slot.orderLimit → иначе SlotFullError (FR-020).
3. Инкрементировать reservedQty для каждого menuItem и reservedCount слота на 1.
4. Создать Order со статусом CREATED, сгенерировать orderNumber (например CP-{YYYYMMDD}-{6-значный random}).
5. Создать OrderItem и подсчитать totalAmount.
6. Создать Notification клиенту: «Заказ #{number} создан».
7. Создать AuditLog action=`order_created`.
8. Вернуть Order с items.

### payOrder(orderId, method: 'ONLINE'|'CASH')
- ONLINE: создать Payment{status=PENDING}, имитировать задержку 2 сек (await sleep(2000)), перевести в SUCCESS, Order.status=PAID. Уведомление «Оплата прошла».
- CASH: создать Payment{status=PENDING}, Order.status=PENDING_PAYMENT. Уведомление «Оплатите на кассе при получении».

### cancelOrder(orderId, actorId)
- Запретить если status ∈ {PAID, READY, ISSUED, CANCELLED} или слот уже начался.
- Декремент reservedQty и reservedCount, Order.status=CANCELLED.
- Notification клиенту, AuditLog.

### issueOrder(orderId, cashierId)
- Если PENDING_PAYMENT — сначала разрешить кассиру вызвать markCashPaid (Payment.status=SUCCESS, Order.status=PAID).
- Order.status=ISSUED, issuedAt=now, issuedById=cashierId.
- Уменьшить availableQty на quantity для каждой позиции (фактическое списание).
- AuditLog action=`order_issued`.

### toggleStopList(menuItemId, value: boolean, actorId) (FR-017/018)
- Установить stopListFlag=value.
- Если true: найти все Order со статусом ∈ {CREATED, PENDING_PAYMENT, PAID} содержащие этот menuItem → создать Notification каждому клиенту: «Блюдо {name} временно недоступно, свяжитесь с кассой».
- AuditLog action=`stop_list_toggled`.

### setSlotLimit(slotId, newLimit, actorId) (FR-019)
- Запретить newLimit < reservedCount.
- AuditLog action=`slot_limit_changed`.

## API endpoints (Next.js route handlers)

Все защищены через NextAuth + RBAC. Возвращают JSON и используют zod для входов.

| Метод | Путь | Роль | Назначение |
|---|---|---|---|
| GET | /api/menu/today | any | Опубликованное меню на сегодня + доступные слоты |
| POST | /api/orders | CLIENT | Создать заказ |
| GET | /api/orders/mine | CLIENT | Список заказов клиента |
| POST | /api/orders/[id]/cancel | CLIENT (owner) | Отмена неоплаченного |
| POST | /api/orders/[id]/pay | CLIENT (owner) | Mock-оплата |
| GET | /api/cashier/search?q=... | CASHIER | Поиск по orderNumber/email/QR |
| POST | /api/cashier/orders/[id]/cash-paid | CASHIER | Принять оплату наличными |
| POST | /api/cashier/orders/[id]/issue | CASHIER | Подтвердить выдачу |
| GET/POST/PUT/DELETE | /api/manager/dishes[/[id]] | MANAGER | CRUD блюд |
| GET/POST/PUT | /api/manager/menu[/[id]] | MANAGER | DailyMenu |
| POST | /api/manager/menu/items/[id]/stop | MANAGER | Toggle стоп-лист |
| GET/POST/PUT/DELETE | /api/manager/slots[/[id]] | MANAGER | CRUD слотов |
| GET | /api/manager/queue | MANAGER | Сводка по слотам на сегодня |
| GET | /api/manager/reports?type=daily|weekly&date=... | MANAGER | Сформировать отчёт |
| GET | /api/notifications | any | Уведомления текущего юзера |
| POST | /api/notifications/[id]/read | any | Отметить прочитанным |

## Acceptance tests (минимум, vitest)

1. client@demo логинится, видит меню, добавляет 2 позиции в корзину, выбирает слот, оформляет онлайн-оплату → заказ становится PAID, reservedQty/reservedCount корректно увеличены.
2. client@demo создаёт заказ с оплатой на кассе → статус PENDING_PAYMENT. Затем cashier@demo находит по orderNumber, принимает оплату → PAID, выдаёт → ISSUED, availableQty уменьшен.
3. client@demo отменяет неоплаченный заказ → CANCELLED, reservedQty/reservedCount возвращены.
4. manager@demo ставит стоп-лист на блюдо, у которого есть активный заказ → клиент получает Notification.
5. Менеджер задаёт orderLimit=1 для слота, 2-й клиент пытается оформить → ошибка SlotFullError, UI показывает «Слот заполнен, выберите другой».
